What makes it unique

このシステムが特別な3つの理由

Git-native State

すべてのデータ、意思決定、学習履歴がGitリポジトリに蓄積されます。LLMのステートレス性を、ファイルシステム＋Gitの差分管理で補完。セッションをまたいでも文脈が失われません。

自己進化するガバナンス

AIがミスをすると自動的にLessonとして記録。信頼度が閾値を超えると、ルール・フック・権限に自動昇格を提案。「自動化が自動化を改善する」メタレイヤーが稼働しています。

非エンジニアが構築

プログラミング経験のないBPO経営者が、CLAUDE.md・Hooks・Skills・Subagentsという「AIのための開発環境」を直接構築。ドメインエキスパートがAIを自在に扱う実証です。

Deep Dive

4つの軸で見る代表事例

Agent Architecture

ナレッジ統合検索エンジン

社内7つのデータベース（3,200件超）を「一つの脳」として横断検索。質問タイプに応じて検索優先度を自動調整し、5段階の推論で根拠付きの回答を生成します。

Before 複数のDBを手動で検索、情報の突合に30分以上。「誰に聞けばいいか」がわからない。
After 1つの質問で7DB横断検索。信頼度スコア付きの回答が30秒で返る。根拠リンク付き。
Claude AI Notion API 多段推論 信頼度スコアリング

Human-in-the-Loop

多層防御のAIガバナンス

AIの動作を4層で制御：ルール定義（CLAUDE.md）→ 権限管理（settings.json）→ 実行時バリデーション（Hook）→ 事後監査（活動ログ）。タスクの性質に応じて自律度を段階的に設計。

Before AIの誤操作リスクが高く、重要データへの書き込みに不安。都度確認が必要。
After 権限を細粒度で制御。読み取りは自由、書き込みは許可リスト制。バリデーション通過率99%超。
Hooks Permission Rules バリデーション 監査ログ

Data Pipeline Orchestration

月次給与突合チェック完全自動化

複数クライアントの給与データを自動突合。勤怠API → 給与API → スプレッドシート比較 → 差分検出 → 結果レポート → クラウドアップロードまで、完全自動で実行します。

Before 手作業で突合。175名分のチェックに月5-10時間。見落としリスクあり。
After 1コマンドで完了。NG/WARN/OK判定を自動色分け。ヒューマンエラーゼロ。
勤怠API 給与API Google Drive Excel

Self-Evolving System

学習ループによる自己改善

セッションごとにミス・修正・発見を「Lesson」として自動抽出。信頼度スコアが閾値を超えると、ルール・フック・スキルへの昇格を自動提案。人間が承認して適用するサイクルが回り続けます。

Before 同じミスを繰り返す。改善は手動で気づいた時だけ。属人的。
After ミス→記録→分析→ルール化が自動。同じエラーは2度と起きない仕組み。
/learn /evolve 信頼度管理 YAML/MD

Getting Started

再現するための3ステップ

1

CLAUDE.mdを書く

AIの役割、禁止事項、データ構造の規約を1ファイルに定義。これがシステムの「憲法」になります。最初は10行で十分。

# CLAUDE.md
あなたは{役割}です。
## 絶対ルール
- {禁止事項}
## データ構造
- data/tasks/ — タスク管理
- data/logs/ — 活動ログ
2

Hooksで自動化する

セッション開始・終了、ツール実行前後に自動処理を差し込みます。バリデーション、ログ記録、引き継ぎ生成を自動化。

// settings.json
"hooks": {
  "SessionStart": [{
    "command": "briefing.sh"
  }],
  "PreToolUse": [{
    "command": "validate.sh"
  }]
}
3

Skillsを育てる

繰り返す作業をスキル（スラッシュコマンド）化。1つずつ追加して、システムを有機的に成長させます。学習ループで品質は自動向上。

# .claude/skills/payroll.md
---
name: payroll-check
description: 月次給与チェック
---
## 手順
1. Driveから原本取得
2. API でデータ取得
3. 突合チェック実行
