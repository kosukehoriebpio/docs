月次勤怠締めワークフロー

毎月末〜翌月9日の約10日間で実施する勤怠締め業務の全体フローです。

月末5日前 月末 翌月3日 翌月7日 翌月9日 | | | | | +-- [Phase 1] ----------+ | | | | リマインド | | | | | | | | | | +-- [Phase 2] -----+ | | | | データ確認・突合 | | | | | | | | | | +-- [Phase 3] -----+ | | | | 上長確認・判断 | | | | | | | | | | +-- [Phase 4] --+ | | | | 最終チェック | | | | | 承認完了 |

Phase 1: リマインド（月末5営業日前〜月末）

未提出・未申請・未承認を事前に解消し、月末時点でのデータ完備を目指す。

タスク対象システムアクション頻度
勤怠提出リマインド（#1）全従業員MF勤怠 / エアシフト未提出者リスト抽出 → Slack通知月末5日前・3日前・前日
有給事前申請リマインド（#1）全従業員MF勤怠未申請の有給使用を検知 → Slack通知同上
経費リマインド（#11）全従業員MF経費下書き・未承認 → 本人&承認者にSlack通知同上
並行処理: 勤怠リマインドと経費リマインドは同時に実施可能。MF勤怠系とエアシフト系（歯科・薬局）は別担当で並行可。

Phase 2: データ確認・突合（月末〜翌月3日頃）

勤怠データの正確性を確認し、給与計算メモへの転記を完了する。

共通タスク（全事業部）

タスクシステムアクション
有給突合（#2）MF/エアシフト ↔ 給与計算メモ有給申請数 = 記載数を確認。不一致 → 修正
有給日数転記（#3）MF/エアシフト → 給与計算メモアルバイトの有給日数を転記
欠勤日数転記（#4）MF/エアシフト → 給与計算メモ欠勤日数を転記
休憩・休暇確認（#8）MF/エアシフト休憩時間・休暇時間の異常値チェック
残業時間集計（#10）MF勤怠残業80h/100h超過者を特定
通勤手当判定（#9）MF勤怠1割有給等・9割出社の判定
経費締め（#12）MF経費全承認確認後に月次締め
