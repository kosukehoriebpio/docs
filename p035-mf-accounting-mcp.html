OAuth認証フロー

MCPサーバーはOAuth 2.0 Authorization Code Flowで認証します。トークン有効期限は1時間、リフレッシュトークンは発行されません。

authorize
認証URL発行
→
ブラウザ認証
MFログイン+許可
→
exchange
コード→トークン
→
API利用
1時間有効

認証の注意点

  • コールバックページに表示されるのは「認可コード」であり、アクセストークンではない
  • mfc_ca_exchange で認可コードをアクセストークンに交換する必要がある
  • リフレッシュトークンは発行されないため、1時間ごとに再認証が必要

BPO業務で活用できるワークフロー

#ワークフロー使用ツール説明
1 月次チェック getReportsTransitionPL 前月比較で異常値を自動検知。売上・経費の急増減をアラート
2 仕訳一括取得 getJournals + pagination 月次仕訳データの全件取得・外部ツールとの照合
3 定型仕訳投入 postJournals 毎月の家賃・給与等の定型仕訳を自動投入
4 仕訳修正 putJournals エラー仕訳の一括修正（金額・税区分・摘要）
5 試算表確認 getReportsTrialBalance* 月次〆後のBS/PL試算表を取得・検証

制約・注意事項

項目状況BPO業務への影響
トークン有効期限 1時間 大量仕訳処理時に再認証が必要。リフレッシュトークンなし
仕訳一括投入 1件ずつ バルクAPIなし。並列呼び出しで対応可能か要検証
仕訳削除 APIなし Web UIからの手動削除が必要
複合仕訳 制約あり 各行の借方・貸方両方にvalue必須（0は不可）
消費税 自動計算 税区分指定で仮払/仮受消費税が自動分離。インボイス区分も自動付与
複数事業者 未検証 Phase 4で検証予定。セッション再設定が必要な見込み
英日辞書 データ空 alpha版のため未実装。今後データ追加の可能性
