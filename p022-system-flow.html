SR1 勤怠チェック自動化システム

p022（グループSR1 労務業務効率化）Phase 2 設計 — 最終更新: 2026-04-02

このページについて

  • 月次給与締め作業に関わる28項目のチェックを自動化するシステムのフロー全体像
  • MFクラウド勤怠パートナーAPI（OAuth 2.0）によるデータ自動取得が主経路（2026-04-02確定）
  • 全てルールベース（AI/LLM不使用）。判定ロジックは数値比較・閾値判定・データ突合のみ
  • 出力先: 給与計算メモSS（自動書き込み）/ Slack Bot（通知・確認）/ GitHub Pages（進捗ボード）

エラー種別対応
429 Rate LimitRetry-After秒数だけ待機 → リトライ。同時リクエスト最大3並列
5xx Server Error指数バックオフ（1s → 2s → 4s → 8s）最大3回リトライ
401 UnauthorizedRefresh Tokenで自動リフレッシュ。失敗時は管理者にSlack通知
50%以上取得失敗CSVフォールバック自動切替 + Slack #グループ労務 に障害通知
従業員単位の失敗スキップしてエラーログに記録。他の従業員は続行
