sauna-seo-writer セットアップガイド

Claude Code プラグイン — Japan Sauna SEO記事生成ツール v1.0.0

このプラグインでできること

  • SEO記事の下書き生成 — キーワード指定で10,000〜18,000字の長文記事を自動生成
  • 品質チェック — テンプレート準拠・文字数・CTA・内部リンク等を自動検証
  • meta description 生成 — GSCデータに基づいた最適化済みmeta descriptionを一括生成
  • GSC連携 — Google Search Console のキーワードデータを動的に取得・分析

概要

japansauna.jp の既存17記事を詳細分析して得られた成功パターン（タイトル形式・見出し構成・独自アンケート・CTA配置・内部リンク戦略・権威引用）をテンプレート化し、Claude Codeのスキルとして実行可能にしたプラグインです。

3つのスラッシュコマンド

コマンド 機能 入力 出力
/sauna-seo-writer:generate-article 記事下書き生成 キーワード or 記事ID MD形式の記事（10,000〜18,000字）
/sauna-seo-writer:validate-article 品質チェック MDファイルパス チェックリスト結果 + 修正提案
/sauna-seo-writer:generate-meta meta description生成 記事スラッグ or all 120〜160字のmeta description

対応する記事タイプ

タイプ 目標文字数 特徴
効果・健康系 15,000〜18,000字 PubMed等の学術引用5本以上必須、YMYL対応
費用・設備系 10,000〜14,000字 費用比較テーブル+FAQ必須
選び方・比較系 10,000〜12,000字 比較テーブル必須、Japan Saunaは公平に掲載
解説・How-to系 10,000〜12,000字 ステップ形式の手順解説
法律・規制系 8,000〜10,000字 条文引用・免責注意書き必須

セットアップ（3ステップ）

1. クローン
→

3. プラグイン読込
1 リポジトリをクローン

ターミナルで以下を実行します。

git clone https://github.com/kosukehoriebpio/claude-plugin-sauna-seo-writer.git
cd claude-plugin-sauna-seo-writer
private リポジトリです。 クローンにはGitHubアカウントへのアクセス権が必要です。権限がない場合は管理者（堀江）に依頼してください。
2 GSC MCP を接続 推奨

Google Search Console MCP サーバーを接続します。japansauna.jp のGSCプロパティへのアクセス権が必要です。

利用中のGSC MCPサーバーの種類に合わせてコマンドを実行してください。

# 例: npm パッケージの場合
claude mcp add gsc -- npx <gsc-mcp-server-package-name>

# 例: ローカルサーバーの場合
claude mcp add gsc -- node /path/to/gsc-mcp-server/index.js

初回接続時にGoogleアカウントでの認証が求められます。認証後、japansauna.jp のプロパティにアクセスできることを確認してください。

確認ポイント: Claude Code を起動して /mcp を実行し、gsc が接続済みになっていればOKです。
3 Claude Code にプラグインを読み込む

以下の3つの方法から選択してください。

方法A: --plugin-dir フラグ（セッションごと）

# クローンしたディレクトリの親から実行する場合
claude --plugin-dir ./claude-plugin-sauna-seo-writer

# クローンしたディレクトリ内から実行する場合
claude --plugin-dir .

方法B: シェルエイリアス（推奨）

毎回 --plugin-dir を指定する手間を省きたい場合は、シェルエイリアスを設定します。

# .bashrc や .zshrc に追加
alias claude-sauna='claude --plugin-dir ~/claude-plugin-sauna-seo-writer'

以降は claude-sauna で起動するだけでプラグインが自動読み込みされます。

方法C: skills / commands を手動コピー

--plugin-dir が利用できない場合の代替手段です。

# skills をコピー
cp -r skills/* ~/.claude/skills/

# commands をコピー
cp -r commands/* ~/.claude/commands/
注意: この方法では ${CLAUDE_PLUGIN_ROOT} が解決できないため、各 SKILL.md 内のパスを絶対パスに書き換える必要があります。

動作確認

Claude Code を起動し、以下のコマンドで動作を確認してください。

# ヘルプを表示
/sauna-seo-writer:sauna-seo

# サンプル記事の品質チェックを実行してみる
/sauna-seo-writer:validate-article samples/01-madori-dosen.md
成功判定: コマンドが認識され、チェック結果が表示されればセットアップ完了です。

使い方

記事の下書き生成

キーワードまたは記事IDを指定して記事を生成します。

# キーワード指定
/sauna-seo-writer:generate-article "自宅サウナ 間取り"

# 記事ID指定（keyword-candidates.json で定義済み）
/sauna-seo-writer:generate-article 2A-2

# 引数なし → 未着手の記事候補一覧を表示
/sauna-seo-writer:generate-article

記事ID一覧（フェーズ2A: 最優先）

ID テーマ 主要KW 記事タイプ 状態
2A-1 自宅サウナの間取り・動線設計ガイド 自宅サウナ 間取り How-to 下書き済
2A-2 マンション・賃貸でサウナを導入する全注意点 自宅サウナ マンション How-to 未着手
2A-3 家庭用サウナメーカー比較・選び方ガイド2026年版 家庭用サウナ メーカー 比較 比較 未着手

全16記事のIDは data/keyword-candidates.json に定義されています。

記事の品質チェック

/sauna-seo-writer:validate-article ./output/drafts/mansion-chintai.md

以下の観点で自動検証し、合格/不合格と修正提案を出力します。

  • タイトル形式（パターンA/B準拠、30〜45字）
  • 見出し構成（H2: 4〜7本、H3: 6〜13本）
  • 文字数（記事タイプ別の基準範囲）
  • 独自アンケート調査セクションの有無
  • CTA配置（最低3箇所、推奨4箇所）
  • 内部リンク（ハブ記事への必須リンク）
  • 画像プレースホルダー（7〜12枚、全alt属性付き）
  • meta description（120〜160字）
  • 記事タイプ別の追加要件（学術引用/FAQ/比較表/法律引用）

meta description 生成

# 単一記事
/sauna-seo-writer:generate-meta MistSauna-Home

# 全記事一括 → output/drafts/meta-descriptions.md に出力
/sauna-seo-writer:generate-meta all

データの更新

記事を新しく公開した場合は、以下の2ファイルを更新してください。

  1. data/existing-articles.json — 新記事のスラッグ・URL・キーワードを追記
  2. data/internal-link-map.json — 新記事への内部リンク先を定義

最新データの取得は以下のコマンドで行います。

cd claude-plugin-sauna-seo-writer
git pull

トラブルシューティング

「GSC MCPサーバーが接続されていません」

GSC MCP未接続でも記事生成は動作します。data/keyword-candidates.json の過去データが代替として使用されます。GSC接続が必要な場合は「セットアップ > Step 2」を参照してください。

スキルが認識されない

--plugin-dir の指定パスが正しいか確認してください。ディレクトリ内に .claude-plugin/plugin.json が存在するか確認します。

ls .claude-plugin/plugin.json

記事IDが見つからない

data/keyword-candidates.json に登録されている記事IDを確認してください。IDの形式は 2A-1, 2B-3 等です。

出力ディレクトリのエラー

output/drafts ディレクトリは .gitignore に含まれるため、クローン直後は存在しません。スキルが自動作成しますが、手動で作成する場合:

mkdir -p output/drafts

git clone でアクセス拒否される

private リポジトリのため、GitHubアカウントへのアクセス権が必要です。管理者（堀江）にリポジトリへの Collaborator 追加を依頼してください。

セキュリティ: このリポジトリは private で運用しています。GSCのアクセストークンはMCP接続時にユーザーごとに認証されるため、リポジトリには含まれません。
