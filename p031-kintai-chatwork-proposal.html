勤怠チェック自動通知のご提案

freee人事労務 × Chatwork連携による勤怠管理の効率化

ご提案の概要

  • freee人事労務のデータをもとに、残業超過・打刻漏れなどの勤怠エラーを自動検知します
  • 検知結果をChatworkの指定チャットに自動で通知します
  • 月末の締め作業前に問題を早期発見し、修正対応の負担を軽減します
  • 36協定の超過回数を自動カウントし、法令リスクを未然に防止します

1. 現状の課題

現在の勤怠チェック業務では、以下のような課題が発生しています。

手作業によるチェックの限界

現在の運用

  • freeeの勤怠データを目視で確認
  • 残業時間の集計・閾値チェックをExcelで手動管理
  • 問題を発見したらChatworkに手動で報告
  • 月末にまとめてチェックするため、修正が集中
  • 36協定の超過回数は過去データを遡って確認
→

導入後のイメージ

  • 勤怠データの取得から通知まで自動で実行
  • 閾値を超えた時点で即座にChatworkへ通知
  • 月中でも途中経過を定期レポート
  • 問題の早期発見・早期対応が可能に
  • 36協定の超過回数を常に自動追跡

現状データから見える状況

いただいた勤怠データ（2025年12月実績）を分析した結果、以下の状況が確認されました。

項目 数値 備考
対象従業員数 87名（在籍者） 34部署・12種の雇用区分
月間残業 45h超 9名 年間休日115日基準
月間残業 42h超 14名 年変形・年間休日105日基準

2. ご提案内容

freee人事労務とChatworkを連携し、以下の機能を提供します。

FEATURE 01

残業超過アラート

設定した閾値（42h / 45h等）を超えた従業員を自動検知し、Chatworkに通知します。月中の途中経過もレポートできるため、超過前の予防的対応が可能になります。

FEATURE 02

打刻漏れ検知

出勤打刻のみで退勤打刻がない、または打刻の整合性が取れないケースを検知します。翌営業日の朝に前日分をまとめて通知するなど、運用に合わせた設定が可能です。

FEATURE 03

36協定カウンター

従業員ごとに36協定の超過回数を自動カウントし、上限（年6回）に近づいた時点でアラートを発信します。年間休日数に応じた閾値（42h / 45h）の自動切替にも対応します。

FEATURE 04

定期レポート

週次または月次で、部署別・従業員別の残業状況サマリをChatworkに自動投稿します。Excel管理台帳と同等の情報を、手作業なしで共有できます。

FEATURE 05

深夜・休日労働の可視化

通常残業だけでなく、深夜労働（22時〜5時）や法定休日労働の発生状況も検知対象にできます。割増賃金の発生を事前に把握し、人件費管理にも活用できます。

通知イメージ

Chatworkへの通知は、以下のようなフォーマットで届きます。

【残業超過アラート】2026年3月 第3週時点

以下の従業員が残業閾値を超過しています。

 ・池田 勇海（工務部）: 46.2h ← 36協定 今期4回目 ※要注意

■ 42h超（年間休日105日）
 ・杉浦 一倫（クラフト部）: 43.8h ← 36協定 今期4回目 ※要注意

 ・河野 愛実（本部/総務）: 3/20 退勤打刻なし
通知先のチャットルーム、通知タイミング（毎日 / 週1回 / 閾値超過時のみ）、通知フォーマットはご要望に応じてカスタマイズ可能です。

freeeのAPI連携について

本システムはfreee人事労務の公式APIを利用します。freee側で「API連携アプリ」の設定を行っていただく必要があります。

必要な設定: freee管理画面の「設定」→「API連携」から、連携アプリの承認を行います。
必要な権限: 勤怠データの読み取り権限（書き込みは行いません）
対応プラン: ベーシックプラン（またはスタータープラン）以上でご利用いただけます

Chatworkの設定について

通知の送信にはChatwork APIを利用します。

必要な設定: 通知用のChatworkアカウントからAPIトークンを発行します
対応プラン: 全プラン対応（ビジネス・エンタープライズプランの場合は管理者の承認が必要です）

4. ヒアリング事項

導入にあたり、以下の項目についてお伺いさせてください。

A. 通知条件について

A-1
残業超過の閾値は現在のExcel管理と同じ設定（45h / 42h）でよいですか？
年間休日数に応じた2段階の閾値設定を想定しています
A-2
閾値超過の「予告通知」は必要ですか？（例: 40hを超えた時点で注意喚起）
超過前に対策を打てるよう、段階的な通知も設定可能です
A-3
打刻漏れの検知は必要ですか？ 必要な場合、どのタイミングで通知しますか？
翌朝一括通知、リアルタイム通知などが選択可能です
A-4
深夜労働・休日労働の通知は必要ですか？
割増賃金の管理に活用できます
A-5
固定残業時間（30h / 40h）の超過も通知対象にしますか？
固定残業超過分の時間外手当の発生を事前に把握できます

B. 通知方法について

B-1
通知先のChatworkチャットルームを教えてください
既存のチャットルーム、または専用ルームの新規作成いずれも対応可能です
B-2
通知の頻度はどの程度を希望されますか？
毎日（朝） / 週1回 / 閾値超過時のみ / 定期レポート＋超過時通知 などから選択
B-3
部署ごとに通知先を分けたいですか？ それとも全社一括でよいですか？
部署管理者ごとに通知を分けることも可能です

C. 対象範囲について

C-1
全従業員が対象ですか？ パート・アルバイトを含みますか？
雇用区分による対象者の絞り込みが可能です
C-2
特定の部署のみを先行導入することは可能ですか？
段階的な導入も対応可能です
C-3
勤怠の締め日はいつですか？（月末 / 15日 / その他）
締め日に合わせた集計期間の設定に必要です

D. 環境について

D-1
freee人事労務のご利用プランを教えてください
ベーシック / スタータープラン以上でAPI連携が利用可能です
D-2
Chatworkのご利用プランを教えてください
ビジネス・エンタープライズプランの場合、管理者によるAPI利用の承認が必要です
D-3
freeeの管理者権限をお持ちの方はどなたですか？
API連携の設定に管理者権限が必要です
