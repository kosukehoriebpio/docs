Invoice Payment Plugin 概要 導入手順 使い方 処理フロー 対応ツール リファレンス FAQ

請求書振込業務プラグイン

経理BPOの請求書→振込業務を標準化・自動化する Claude Code プラグイン

概要

何ができるか

/invoice-payment {クライアント名} の1コマンドで、請求書の収集から振込データ作成、消込確認まで6ステップを一気通貫で実行します。 クライアントごとの業務ルール差分は「リファレンスファイル」で吸収するため、どのクライアントでも同じコマンドで動きます。

129

クライアント対応

Notion顧客マニュアルDBから自動生成されたリファレンスで129社をカバー

9

会計ツール対応

バクラク / MF債務支払 / freee / MF会計 / 弥生 / 奉行 / ICS / TKC / PCA

4

収集ソース

Google Drive / Gmail / バクラクAPI / 手動配置の4つから自動判定

銀行IB操作は絶対に自動化しません。 振込の最終実行は必ず人間がインターネットバンキングで行います。 プラグインが行うのは「振込データ（FBファイル）の作成」と「操作手順の案内」までです。
導入手順

インストール（初回のみ、3分）

前提条件

  • Claude Code がインストール済みであること
  • Python 3.10+（PDF読取に使用）
  • Node.js 18+
  • kosukehoriebpio GitHub Organization のメンバーであること
セキュリティ: プラグインとリファレンスはGitHub private リポジトリで管理されています。Organization メンバーのみがアクセスできます。GitHub CLI の認証がまだの場合は gh auth login を先に実行してください。

Step 1: プラグインのインストール

ターミナルで以下を実行します:

/plugin install invoice-payment@kosukehoriebpio

自動的に以下がセットアップされます:

  • プラグイン本体（スクリプト5本 + SKILL.md）
  • 銀行コードマスタ（bank-codes.json）

Step 2: Python依存パッケージのインストール

pip install pdfplumber

Step 3: 初回実行で自動セットアップ

初回の /invoice-payment 実行時に、129社分のリファレンスが自動でダウンロードされます（git clone）。以降は毎回自動 pull で最新版が取得されます。

これだけで使えます。 特別な設定やAPIキーは不要です（Drive/Gmail自動収集やバクラクAPI連携を使う場合のみ追加設定が必要）。
使い方

基本コマンド

# 全ステップ通し実行（最も一般的）
/invoice-payment ship

# 特定ステップのみ実行
/invoice-payment ship collect    # Step 1: 収集のみ
/invoice-payment ship check      # Step 3: チェックのみ
/invoice-payment ship pay        # Step 4: 振込データ作成のみ

クライアント名の指定

リファレンスファイルのファイル名（slug）で指定します。会社名のうち「株式会社」等を除いた部分です。

会社名コマンド
株式会社SHIP/invoice-payment ship
BIRD INITIATIVE株式会社/invoice-payment bird-initiative
株式会社マル勝高田商店/invoice-payment マル勝高田商店
株式会社Timewitch/invoice-payment timewitch

slug がわからない場合は /invoice-payment（引数なし）で実行すると、対話的にクライアントを選択できます。

実行の流れ

コマンドを実行すると、Claude が各ステップを順番に進めます。途中でユーザーへの確認が入るポイントが3箇所あります:

  1. Step 3 後 — チェック結果の確認（NGがあれば除外するか判断）
  2. Step 4 後 — 振込サマリの確認（金額・件数の最終確認）
  3. Step 5 — 銀行IBでの振込実行（手動操作 → 完了報告）
リファレンスに情報が足りなくても止まりません。 不足している情報（振込元口座、定期取引先リストなど）はその場で質問されます。わからなければ「わからない」と答えれば手動フォールバックに切り替わります。
処理フロー

6ステップの流れ

0

初期化

リファレンスリポの同期（自動pull）→ クライアントリファレンスの読み込み → 作業ディレクトリ作成

1

請求書収集 collect.ts

リファレンスの設定に基づき、Google Drive / Gmail / バクラクAPI / 手動配置 の4ソースから請求書PDFを収集。 収集されたファイルの一覧が _manifest.json に記録されます。

2

請求書読取・データ化 extract.py

PDFのテキストレイヤーから金額・口座・日付等を構造パース（pdfplumber）。 テキストが取れない画像PDFのみ Claude Vision でフォールバック読取。 取引先名 / 金額 / 税額 / 振込先口座 / 明細行 / 登録番号を抽出。

3

内容チェック check.ts

6項目を自動チェック: 二重請求検出 / 金額妥当性 / 取引先マスタ照合 / 支払期日 / 高額アラート / 源泉徴収。 結果は OK / WARN / NG で分類され、NGは除外を提案。

4

振込データ作成 generate-fb.ts

全銀協FBフォーマットの振込ファイルを生成。振込サマリ（Markdown）も出力。 会計ツール別の連携手順（バクラク / MF / freee 等）も自動案内。 生成されたFBファイルのフルパスが画面に表示されます。

5

振込実行（手動）

銀行IBの操作は必ず人間が行います。 Claude は IB の種類と操作手順を案内し、FBファイルのパスと金額を提示します。完了報告を待ちます。

6

消込確認 reconcile.ts

振込結果と請求データを突合し、消込OK / 未消込 / 不一致を検出。 取引先名のファジーマッチ（カナ⇔漢字）に対応。

リファレンス

リファレンスファイルとは

各クライアントの業務ルール（どの会計ツールを使うか、振込元口座はどこか、定期取引先は誰か等）を記録したMarkdownファイルです。 129社分が自動生成済みで、.invoice-payment-references/ に格納されています。

構成（7セクション）

セクション内容自動抽出
0. 会計ツール使用ツール名、API可否済
1. 請求書収集収集方法、DriveフォルダID一部
2. 読取ヒント既知取引先のフォーマット特徴-
3. チェックルール定期取引先リスト、閾値-
4. 振込データ振込元口座、支払サイクル一部
5. 振込実行IB種類、承認フロー一部
6. 消込確認消込方法-
リファレンスの補完は任意です。 空欄のままでもプラグインは動作します（実行時に質問されるか、手動フォールバック）。ただし、定期取引先リストや振込元口座を埋めるほどチェック精度が上がります。

リファレンスの編集方法

# リファレンスリポを開く
cd .invoice-payment-references/

# SHIPのリファレンスを編集
code ship.md

# 編集後、コミット & プッシュで全社員に反映
git add ship.md
git commit -m "update ship reference"
git push
FAQ

Q: プラグインを使うのに特別な権限は必要？

kosukehoriebpio GitHub Organization のメンバーであれば使えます。追加のパスワードやキーは不要です。gh auth login でGitHubにログインしていれば、初回実行時に自動でセットアップされます。Drive/Gmail自動収集を使う場合のみ Google API の認証が別途必要です。

Q: 社外の人がプラグインを使えてしまう？

使えません。プラグイン本体もクライアントデータ（リファレンス）も GitHub private リポジトリで管理されており、Organization メンバー以外はアクセスできません。退職等でOrgから外されると即座にアクセス不可になります。

Q: 新しいクライアントを追加するには？

/invoice-payment 新しいslug で実行すると、テンプレートからリファレンスを作成するか聞かれます。作成後にリファレンスリポにプッシュすれば全社員に反映されます。

Q: 振込データ（FBファイル）はどこに保存される？

作業ディレクトリ .tmp-invoice-payment/{クライアント名}/{年月}/ に保存されます。実行時にフルパスが画面に表示されるので、それをコピーしてエクスプローラーで開いてください。

Q: チェックでNGが出たらどうする？

NG項目を除外して残りを続行するか、全体を中断するかを選べます。除外された請求書は振込データに含まれません。

Q: リファレンスに情報が足りない場合は？

3段階で対応します: (1) リファレンスにあればそのまま使用 → (2) なければその場で質問 → (3) わからなければ手動操作に切替。止まることはありません。

Q: 間違って振込実行してしまったら？

プラグインが直接銀行操作をすることはありません。FBファイルのアップロードと振込承認は全て手動です。銀行IBの承認フローで止めることができます。

Q: 月の途中で追加の請求書が来たら？

同じクライアント・同じ月で再度 /invoice-payment {slug} collect を実行すれば、追加ファイルが収集されます。
