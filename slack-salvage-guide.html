概要 — このツールは何をするのか

/slack-salvage は、あなたが参加している Slack の全チャンネル・DM を横断スキャンして、 「チケットやタスク管理ツールに登録されないまま止まってしまった非公式プロジェクト」を掘り起こすツールです。

こんな状況に心当たりがあれば使ってみてください：
  • 「あの件、誰かやるって言ってたよね？」が追えていない
  • 「やった方がいいよね」で止まったまま数ヶ月経っている
  • Slack でいい議論をしたのに、誰も動かずそのまま消えた
  • 自分が「承知しました」と言ったまま完了報告を忘れていた

セットアップ（初回のみ・約 10 分）

前提： Claude Code がインストール済みであること。Node.js（v18 以上）が使えること。

Step 1 ： Slack App を作成する

  1. https://api.slack.com/apps にアクセスしてログインする
  2. 右上の「Create New App」をクリック
  3. 「From scratch」を選択
  4. App Name に任意の名前（例：Claude Salvage）を入力
  5. Workspace で対象のワークスペースを選択して「Create App」

Step 2 ： User Token Scopes を設定する

左メニューの「OAuth & Permissions」を開き、User Token Scopes セクションに以下の権限をすべて追加します。

Scope用途
channels:historyパブリックチャンネルの履歴を読む
channels:readパブリックチャンネルの一覧を取得
groups:historyプライベートチャンネルの履歴を読む
groups:readプライベートチャンネルの一覧を取得
im:historyDM の履歴を読む
im:readDM の一覧を取得
mpim:historyグループ DM の履歴を読む
mpim:readグループ DM の一覧を取得
search:readメッセージを検索する
users:readユーザー名・表示名を解決する
注意：「Bot Token Scopes」ではなく、「User Token Scopes」に追加してください。Bot では自分の DM やプライベートチャンネルにアクセスできません。

Step 3 ： アプリをインストールして User Token を取得する

  1. 同じページ（OAuth & Permissions）の上部にある「Install to Workspace」をクリック
  2. 権限の確認画面で「許可する」をクリック
  3. インストール後に表示される「User OAuth Token（xoxp- から始まる文字列）」をコピーしておく
Workspace ID の確認方法： ブラウザで Slack を開いたときの URL が https://app.slack.com/client/Txxxxxxxx/... の形式になっています。 T から始まる部分がワークスペース ID です。

Step 4 ： .mcp.json に設定を追記する

作業用のプロジェクトフォルダ（どこでも OK）のルートに .mcp.json ファイルを作成し、以下を記述します。

{ "mcpServers": { "slack": { "command": "npx", "args": ["-y", "@korotovsky/slack-mcp-server"], "env": { "SLACK_USER_TOKEN": "xoxp-ここにStep3でコピーしたトークンを貼り付け", "SLACK_TEAM_ID": "TxxxxxxxxのワークスペースID" } } } }
セキュリティ注意： .mcp.json には User Token が含まれます。このファイルを Git にコミットしたり、他人と共有したりしないでください。.gitignore に .mcp.json を追加することを推奨します。

Step 5 ： スキルファイルを配置する

堀江から受け取った SKILL.md を以下のパスに配置します：

.claude/skills/slack-salvage/SKILL.md

フォルダがない場合は作成してください：

mkdir -p .claude/skills/slack-salvage

Step 6 ： Claude Code を再起動する

設定を反映させるため、Claude Code を一度終了して再起動します。

# Claude Code を終了後、プロジェクトフォルダで再起動 claude
✅ 再起動後に /slack-salvage setup と入力して、セットアップ内容が表示されれば設定完了です。

使い方

基本的な実行方法

Claude Code のチャット画面に以下を入力するだけです：

/slack-salvage

直近 60 日間の全チャンネル・DM をスキャンします。チャンネル数の確認が入るので「y」と答えると実行が始まります。

引数一覧

コマンド動作おすすめの使いどころ
/slack-salvage 直近 60 日間の全チャンネル+DM をスキャン まず試すときはこれ
/slack-salvage days:180 直近 180 日間をスキャン 半年分まとめて掘り起こしたいとき
/slack-salvage channel:general 指定チャンネルのみスキャン 特定チャンネルに絞りたいとき
/slack-salvage dm DM とグループ DM のみスキャン 1対1のやり取りだけ確認したいとき
/slack-salvage status:stalled 中断・自然消滅のみ絞り込み表示 要対応のものだけ見たいとき
/slack-salvage setup セットアップ手順を表示 設定を確認したいとき

実行後のインタラクション

結果が表示されたら、以下の操作が可能です：

入力動作
番号（例：1）そのアイテムの詳細表示・フォローアップ文案の生成

よくある質問

Q. プライベートチャンネルも見られますか？

A. 自分が参加しているプライベートチャンネルは対象になります。自分が入っていないチャンネルは対象外です。

Q. 他の人のDMを読まれますか？

A. 自分が参加しているDM・グループDMのみ対象です。他の人同士のDMは読みません。

Q. Slackのフリープランでも使えますか？

A. 使えますが、フリープランはメッセージ履歴が直近 90 日までです。それ以前のデータは取得できません。

Q. エラーが出た場合は？

A. レート制限エラーが出た場合は少し待ってから再実行してください。Token エラーが出た場合は xoxp- トークンと Workspace ID を再確認してください。

Q. 「あのスレッド」が出てこない場合は？

A. /slack-salvage days:180 で期間を延ばす、または channel:チャンネル名 で特定チャンネルを指定してみてください。

問題が解決しない場合は堀江（kosuke.horie@sevenrich.jp）にお問い合わせください。
